# PRD — 产品需求文档

## 1. 产品概述

这是一个**网络监控 / 保障 / 修复 AI Agent 操作台**，供网络运维人员与 AI Agent 对话，驱动 Agent 执行网络诊断、分析与修复任务，并在右侧面板实时展示分析结果（图像、图表、报告）。

- **系统名称**：网络 Agent 操作台
- **目标用户**：网络运维工程师
- **核心价值**：通过自然语言驱动 Agent 完成网络故障诊断与分析，结果可视化呈现

---

## 2. 页面清单

本系统为单页应用（SPA），仅一个主路由，通过**左侧面板状态切换**区分两个视图。

| 视图名称 | 路由 | 用途说明 |
|----------|------|----------|
| 主工作台 | `/` | 唯一页面，左侧在「会话列表」和「对话界面」之间切换，右侧动态渲染 Agent 结果 |

## 3. 用户流程

### 主流程 1：发起新对话

1. 用户打开系统，左侧默认展示**会话列表**
2. 点击「新建对话」按钮
3. 左侧切换为**对话界面**（聊天模式）
4. 用户在底部输入框输入问题，回车或点击发送
5. Agent 流式返回：依次出现用户气泡 → 步骤卡片 → 结论容器
6. 若 Agent 返回含渲染标识，右侧面板同步展示图像或数据洞察

### 主流程 2：查看历史会话

1. 用户打开系统，左侧展示历史会话列表
2. 点击某条历史会话
3. 左侧切换为对话界面，展示该会话历史消息
4. 可继续提问

### 主流程 3：返回会话列表

1. 在对话界面，点击顶部 ⬅️ 左箭头
2. 或点击左侧最窄导航栏上的会话列表图标
3. 左侧切换回会话列表视图

## 4. 功能模块拆解

### 模块：最左侧导航栏（48px）

- **Logo 区**：品牌标识，固定在顶部
- **导航图标组**：竖排 SVG 图标按钮，hover 显示 tooltip 文字，点击触发对应操作
  - 会话列表图标：点击返回会话列表视图
  - （预留扩展位，如设置、帮助）

### 模块：左侧面板 — 会话列表视图（484px）

- **头部**：标题「会话列表」+ 新建对话按钮
- **会话列表**：每条展示会话标题、最后消息时间、内容摘要
- 点击任意会话 → 进入该会话的对话界面
- 点击新建 → 创建新会话并进入对话界面

### 模块：左侧面板 — 对话界面（484px）

- **顶部栏**：⬅️ 返回箭头 + 当前会话标题
- **消息滚动区**：纵向排列，包含：
  - 用户提问文本气泡
  - 步骤卡片（可折叠，展开后显示子步骤时间轴）
  - 结论容器（思考过程可折叠，最终结论常驻）
- **底部输入区**：悬浮输入气泡（452×96px），含深度思考按钮（假，88×28px）

### 模块：右侧展示面板（1354px）

- **空状态**：无内容时展示引导提示
- **场景一 — 图像展示**：大图 + 标题 + 文字结论
- **场景二 — 数据洞察**：多图表网格 + 各图文字结论 + 完整 Markdown 报告

---

## 5. 状态与边界处理

| 场景 | 处理方式 |
|------|----------|
| 会话列表为空 | 展示空状态插画 + 「新建第一个对话」引导按钮 |
| Agent 流式响应中 | 步骤卡片逐步出现，展示 loading 动画，禁用输入框 |
| 接口报错 / 流中断 | 在消息区展示错误提示卡片，输入框恢复可用 |
| 右侧图像加载失败 | 展示错误占位图 + 提示文案 |
| 步骤卡片子步骤为空 | 展示「暂无子步骤」占位 |
| 深度思考按钮点击 | UI 状态切换（激活/未激活），不与后端交互 |
